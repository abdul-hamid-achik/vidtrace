# Documentation Site Notes

The current docs are plain Markdown so they can move into a static documentation site later.

## Suggested Navigation

1. Overview: `docs/index.md`
2. Install: `docs/INSTALL.md`
3. Usage: `docs/USAGE.md`
4. Analysis: `docs/ANALYSIS.md`
5. Studio: `docs/STUDIO.md`
6. CLI Reference: `docs/CLI_CONTRACT.md`
7. Artifact Schema: `docs/ARTIFACT_SCHEMA.md`
8. Testing: `docs/TESTING.md`
9. Release: `docs/RELEASE.md`
10. Architecture: `docs/ARCHITECTURE.md`
11. Roadmap: `docs/ROADMAP.md`
12. Agent Prompt: `prompts/analyze-bundle.md`

## Site Requirements

- Keep docs source in Markdown.
- Keep command examples copy-pasteable.
- Keep install instructions separate from development instructions.
- Treat `CLI_CONTRACT.md` and `ARTIFACT_SCHEMA.md` as reference pages.
- Treat `STUDIO.md` as a task guide for human review.
- Do not publish local media fixtures or generated artifact bundles.

## Next Step

A small docs site can be added when the navigation is stable. The site should reuse these files instead of duplicating content.
